안녕하세요. 프로젝트 내용 꼼꼼히 확인했습니다.

배드민턴 기반 스포츠 리그 시스템으로 회원관리, 대회관리, 랭킹관리, 대회운영 기능을 갖춘 하이브리드 앱(iOS/Android)을 구축하는 프로젝트로 이해했습니다.

핵심 요구사항을 정리하면:

경기 일정 관리 — 관리자 수기 경기 일정 등록, 프로/아마추어/학생 카테고리 분류
경기 신청/조회 — 사용자 일정 선택 및 신청, 결제 필수 프로세스
카테고리별 인증 — 학생(나이 인증), 프로(선수증 제출), 관리자 승인 절차
이의제기 — 거절 시 이의제기 기간 내 재심사 요청 가능
대진표 — 진행중 경기 실시간 현황 조회, 토너먼트 승자에 따른 대진표 실시간 업데이트
랭킹 시스템 — 카테고리별 선수 랭킹, 승/패/승률/포인트 기반 순위 산정
결제 연동 — 참가비 결제(카드/계좌이체/카카오페이), 환불 처리
관리자 시스템 — 대회 CRUD, 신청 승인/거절, 시드 배정, 대진표 생성/관리, 결과 입력, 결제/환불 관리

유사한 커머스 플랫폼을 기획, 디자인, 개발까지 1인 풀스택으로 수행한 경험이 있습니다. 토스페이먼츠 PG 결제 실전 연동, PostgreSQL 기반 DB 설계, 소셜 로그인/알림톡 연동 등 본 프로젝트의 핵심 기능과 직접적으로 연관됩니다.

구현 계획:
1단계(04/30~05/13, 14일) — 기획/설계: 요구사항 정의서, DB 스키마, API 설계, 와이어프레임
2단계(05/14~05/17, 4일) — 환경 세팅: Supabase 프로젝트 생성, Expo 초기화, CI/CD 구성
3단계(05/18~05/31, 14일) — 인증/회원: 소셜 로그인(카카오/네이버/구글), 회원가입, 카테고리 인증
4단계(06/01~06/14, 14일) — 대회 관리: 대회 CRUD, 경기 일정표, 카테고리 분류, 상태 워크플로우
5단계(06/15~07/05, 21일) — 핵심 기능: 경기 신청, PG 결제 연동, 토너먼트 대진표, 실시간 업데이트, 랭킹
6단계(07/06~07/19, 14일) — 관리자/알림: 관리자 대시보드, 이의제기, 푸시 알림(FCM), 결제/환불 관리
7단계(07/20~07/26, 7일) — QA/배포: 통합 테스트, 디바이스 테스트, 앱 빌드, 스토어 등록

기술스택: React Native(Expo) + Supabase(Auth/DB/Storage/Realtime) + PostgreSQL + 토스페이먼츠
