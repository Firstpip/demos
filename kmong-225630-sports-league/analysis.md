# 프로젝트 분석: 스포츠리그 경기신청 및 대진표 제공 플랫폼 앱

## 프로젝트 개요

- 프로젝트명: 스포츠리그 경기신청 및 대진표 제공 플랫폼 앱 개발
- 의뢰인: 기업 (법인/개인사업자/예비창업자)
- 예산: 1,500만원
- 기간: 3~4개월 (2026-04-30 ~ 2026-07-31)
- 의뢰 범위: 턴키 (기획/디자인/개발/퍼블)
- 플랫폼: iOS + Android 하이브리드 앱 + 마켓 등록
- 제안 수: 17건

## 요구사항 목록

### 명시된 요구사항 (원본 순서)

1. 스포츠 경기 일정표
   - 관리자의 수기 경기 일정 등록
   - 프로/아마추어/학생 카테고리 분류 (프로는 추후 오픈)

2. 경기 신청 및 조회
   - 사용자의 일정표 선택 및 신청
   - 신청 과정 중 결제 필수
   - 카테고리별 인증 (학생-나이 인증, 프로-선수증 제출 등)
   - 관리자 승인 절차
   - 거절 시 이의제기 기간 내 재심사 요청 가능

3. 진행 중 경기 현황 및 대진표
   - 진행 중인 경기 현황 조회
   - 선수 개인별 참가 경기 정보 조회
   - 토너먼트 승자에 따른 대진표 실시간 업데이트

### 암묵적 요구사항

4. 회원 관리
   - 회원가입/로그인 (소셜 로그인)
   - 사용자 프로필 (선수 정보, 프로필 사진)
   - 카테고리별 인증 상태 관리

5. 관리자 기능
   - 회원 관리 (목록, 인증 승인/거절)
   - 대회 관리 (생성, 수정, 삭제)
   - 경기 결과 입력
   - 신청 승인/거절 처리
   - 이의제기 관리

6. 랭킹 시스템
   - 카테고리별 랭킹
   - 승/패/승률 기반 순위

7. 마이페이지
   - 참가 이력 (과거/현재 대회)
   - 전적 관리
   - 결제 내역

8. 알림
   - 푸시 알림 (경기 일정, 승인/거절, 대회 안내)

9. 기타
   - 향후 영문 버전 및 해외 마켓 등록 고려 (현재 스코프 외)
   - 추후 고도화 대비 확장 가능한 구조

| 영역 | 기술 | 선정 이유 |

## 리스크

1. **결제 연동**: PG사 계약 및 심사 기간 필요 (1~2주). 의뢰인 사업자등록 필요
2. **스토어 등록**: Apple 심사 기간 불확실 (1~7일), 개발자 계정 필요
3. **실시간 대진표**: 동시 접속자 많을 경우 Supabase Realtime 성능 확인 필요 (MVP 수준에서는 문제 없음)
4. **카테고리 인증**: 나이 인증 방식 (본인인증 API vs 자체 신고) 의뢰인과 협의 필요
