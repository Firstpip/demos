## 페르소나

### P-1: 활동 중인 프리랜서 댄서 (20~30대)
- 동기: 자신의 장르·경력에 맞는 공연·프로젝트 기회를 정기적으로 확보
- 통증점: 일반 채용 플랫폼은 직군·JD가 맞지 않고 단발성 프로젝트·공연 모집이 노출되지 않음. 인스타 DM·지인 소개 의존
- 기대 결과: 프로필·영상·경력을 한 번 등록하면 적합 공고 알림이 오고 바로 지원 가능
- 관련 요구사항: R-02, R-03, R-06, R-08, R-09, R-10

### P-2: 공연·프로젝트 섭외 담당자 (기업회원, 엔터테인먼트·광고·축제 기획)
- 동기: 단기 캐스팅·프로젝트에 맞는 검증된 예술인을 빠르게 찾아 섭외
- 통증점: 에이전시 수수료 부담, 포트폴리오·활동 이력 확인 경로 분산
- 기대 결과: 카테고리·경력 조건으로 필터링 후 지원자를 비교·연락
- 관련 요구사항: R-01, R-02, R-03, R-04, R-08

### P-3: 플랫폼 운영 관리자 (의뢰사 내부)
- 동기: 악성 공고·스팸 회원 차단하며 공고 품질 유지
- 통증점: 승인 대기 적체, 위반 게시물 대응 수단 부재
- 기대 결과: 대시보드에서 회원·공고·게시글을 일괄 검토·승인·삭제
- 관련 요구사항: R-05, R-11, R-13

- [ ] R-03 프로필 입력 — 이름·연락처·장르·경력·포트폴리오 링크·프로필 이미지 (근거: 원문 "프로필 입력")
- [ ] R-04 채용공고 CRUD — 기업회원이 공고 등록/수정/삭제. `/jobs/new`는 jobType + 장르 셀렉트 분리 + jobType 변경 시 동적 추가 필드(레슨: 강사 자격·요일·학원 위치·시범 수업 / 공연: 일시·리허설·공연장·의상 / 행사: 일자·드레스 코드·브리핑·식사 / 프로젝트: 산출물·저작권·NDA·활용 범위)를 자동으로 갈아끼움. (근거: 원문 "등록·수정/삭제")
- [ ] R-05 공고 카테고리 분류 — RFP 명시 두 축. (1) jobType: 레슨/공연/행사/프로젝트 — `/jobs` 상단 4탭 + 카드·상세 배지 + `/jobs/new` 첫 셀렉트로 필수 선택. (2) 장르: 댄스/음악/연기/모델/서커스/마술/성우/기타 — 사이드바 필터 + 카드 이중 배지. 추가로 고용형태·지역·경력은 사이드바 필터에서 보조 축. (근거: 원문 "카테고리(레슨/공연/행사/프로젝트)" + "카테고리 분류")
- [ ] R-06 채용공고 리스트/상세 페이지 — 리스트에서 필터·정렬, 상세에서 지원 CTA (근거: 원문 "리스트/상세 페이지")

## 기술스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 14 (App Router) | SSR로 SEO 유리(R-19), 공고 상세 페이지 검색 유입 지원, React 생태계 |
| 인증 | Supabase Auth | 이메일·OAuth 제공, 소셜 로그인 확장 용이, RLS로 권한 분기(R-18) 구현 쉬움 |
| DB | Supabase Postgres | 관계형 + full-text search 기본 제공 (R-08), 추후 확장 용이(R-13) |
| 파일 저장 | Supabase Storage | 프로필·포트폴리오 이미지(R-03, R-16), 인증과 스토리지 일원화 |
| 스타일링 | TailwindCSS + shadcn/ui | 반응형 유틸리티(R-12), 컴포넌트 일관성, MVP 속도 |
| 이메일 | Resend | 회원가입·알림(R-20), 무료 티어 월 3,000통 |
| 배포 | Vercel | Next.js 최적 연동, 프리뷰 배포, 도메인 연결 간단(R-14) |

선정 근거: 인프라 제약(cron·장시간 API·서버사이드 callback)이 MVP 범위에 없음. Supabase가 auth·DB·storage를 통합 제공하여 1인 바이브코딩 공수 최적화. 데모는 Vercel 배포로 시연하되, 의뢰사 인수 시 도메인·명의를 의뢰자 계정으로 이관 가능.

대안 고려: Supabase 대신 Neon + NextAuth + S3 조합은 통합 비용이 크므로 MVP 예산·기간에 부적합.
