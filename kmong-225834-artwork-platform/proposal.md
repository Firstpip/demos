[제안 내용]
안녕하세요. ARTWORK 예술인 채용 공연 협업 플랫폼 MVP 제안드립니다.

실제 동작 데모 URL과 제안서 PDF를 함께 첨부했습니다. 데모에서 시연 시나리오대로 핵심 기능을 직접 확인하시고, PDF에서 요구사항별 구현 방식과 일정 상세를 확인하실 수 있습니다.

프로젝트 이해
ARTWORK는 댄서를 비롯한 프리랜서 예술인이 자신의 장르와 경력에 맞는 공연, 레슨, 행사, 프로젝트 기회를 정기적으로 발견하고, 기업과 학원, 기획사가 검증된 예술인을 빠르게 섭외하도록 설계된 MVP 채용 협업 플랫폼입니다. 일반회원, 기업회원, 운영 관리자 3축 권한 모델 위에 카테고리, 검색, 지원 루프를 갖춘 구인구직과 모집 게시판 통합 구조를 목표로 합니다. 핵심 페르소나는 정기 공연 기회를 찾는 프리랜서 댄서와 개막 2주 전 캐스팅이 급한 공연기획 섭외 담당자입니다.

문제와 원인
일반 채용 플랫폼에는 단발성 공연, 행사, 프로젝트 모집이 노출되지 않고 직군 분류도 맞지 않아, 예술인은 인스타 DM과 지인 소개에 의존해 기회 탐색에 시간을 낭비합니다. 기업과 학원은 에이전시 수수료 부담과 분산된 포트폴리오 확인 경로 때문에 짧은 일정 안에 적합 후보를 찾기 어렵습니다. 결국 양측 모두 검증된 만남을 빠르게 만드는 공통 인프라가 비어 있습니다.

해결 접근
공고를 레슨, 공연, 행사, 프로젝트 4종 jobType으로 분리하고 장르 8종을 보조 축으로 두어 검색 자동완성, 사이드바 필터, 등록 폼이 하나의 어휘로 연결되도록 설계했습니다. 예술인 프로필에는 영상과 경력 타임라인을 우선 배치해 섭외 판단 시간을 줄이고, 운영자가 신규 공고와 신고를 일일 처리 큐에서 일괄 검토하도록 구성했습니다. Next.js와 Supabase 기반으로 인증, DB, 스토리지, 알림을 단일 스택으로 묶어 38일 안에 핵심 루프를 안정적으로 닫습니다.

근거와 수치
공수는 33일로 산정되며 전통 공수 11주 대비 0.4배 수준입니다. 외부 API 연동 비중이 19%로 일정 리스크가 낮고, 제시 기간 38일 대비 5일 여유가 확보되어 MVP 핵심 루프(가입, 공고, 지원, 승인)에 범위를 고정하면 안정적으로 마감 가능합니다. 사전 제출한 데모는 Next.js 16과 Tailwind 4 조합으로 73 페이지를 빌드해 사용자 13개와 관리자 7개 라우트가 동작하며, 시연 시나리오 5종을 Puppeteer로 종주 검증하고 콘솔 에러 0건을 유지하고 있습니다. 유사 경험으로 패키지 디자인 에디터 쇼핑몰을 기획, UI/UX, 프론트엔드, 백엔드, 인프라까지 6개월 단독 수행한 사례를 보유하고 있어 멀티 페르소나 권한 분기, 관리자 콘솔, 이벤트 큐 기반 알림을 그대로 이식할 수 있습니다.

구현 계획
총 38일로 진행합니다. 1주차 초기 세팅과 ERD, Supabase 인증 골격, 2주차 사용자 영역 회원 가입과 프로필, 공고 리스트 및 상세, 3주차 공고 등록과 지원 루프, 프로젝트 모집 게시판, 4주차 관리자 콘솔과 카테고리 관리, 5주차 검색 자동완성, 신고 처리, Resend 이메일 연동, 6주차 QA, 반응형, SEO, 배포와 인계 문서 순으로 마무리합니다. 주차별 상세 일정은 첨부 PDF에 담았습니다.

리스크와 완화
첫째, 예산 300만원 대비 회원 2종, 공고, 게시판, 관리자, 신고까지 범위가 넓어 일정 압박이 큽니다 (K-01, 레벨 고). MVP 범위를 가입과 공고 CRUD, 지원, 카테고리, 검색, 관리자 핵심 9개 기능으로 고정하고, 알림 고도화와 사업자번호 외부 검증은 Phase 2로 분리해 마감 안정성을 확보합니다.
둘째, 확장 가능한 구조 요구의 기준이 모호해 운영 단계에서 분쟁 소지가 있습니다 (K-02, 레벨 중). 초기 ERD에 jobType 추가 필드, 카테고리 마스터, 신고와 알림 채널, 감사 로그, 검색 인덱스, RBAC, i18n 7개 확장 슬롯을 선제 반영하고 인수인계 문서에 확장 절차와 마이그레이션 가이드를 명시합니다.
