# 프로젝트 분석: 개인 건강 데이터 맞춤형 건기식 추천/상담 플랫폼

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | DTC 유전자 및 건강검진 검사 기반 개인형 건강기능식품 추천 플랫폼 |
| 서비스명 | 에너링거(ENER RINGER) (가칭) |
| 의뢰사 | HL Science (에이치엘사이언스) / E커머스 1팀 |
| 예산 | 8,000만원 (세금계산서 발행) |
| 기간 | 2026-04-13 ~ 2026-09-12 (약 5개월, 약 110 영업일) |
| 진행 방식 | 외주 (턴키 구축) |
| 도메인 | genetichealth.co.kr |
| 레퍼런스 | 핏타민(fitamin.kr) — 기능/구조 유사, 디자인은 다르게 |
| 기획서 | 19페이지 기획서 첨부 (v1.0 초안, 2026.03) |
| 제안 현황 | 40건 접수 |

**핵심 컨셉**: 의사 및 전문의가 내방 환자에게 DTC 유전자 검사를 권유하고, 검사 결과 + 건강검진 데이터를 기반으로 맞춤형 건기식을 추천·판매하는 B2B2C 이커머스 플랫폼. 전문가에게 R/S(Revenue Share) 방식 리워드 제공.

## 2. 요구사항 (원본 순서)

### 2.1 데이터 연동 및 분석 알고리즘
1. **DTC 데이터 연동**: 자사 DTC 서비스 결과값 API 연동 및 시각화 (12개 유전자 마커: FTO, TCF7L2, APOE, CYP1A2, VDR, MTHFR, SOD2, LCT, ACTN3, IL6, ALDH2, COMT)
2. **건강검진 데이터 불러오기**: 국민건강보험공단 CODEF API(마이데이터) 연동 (13개 검진항목: 공복혈당, 총콜레스테롤, 중성지방, 간기능 AST/ALT, 혈압, 헤모글로빈, 비타민D, BMI, eGFR, 골밀도, 장건강)
3. **심층 설문 시스템**: 성별, 연령, 생활습관, 복용 약물, 알레르기 등 다단계 설문 로직
4. **AI 추천 엔진**: 검진 수치 + 유전자 취약성 + 설문 결과 종합 → 부족 영양소 도출 → 최적 제품 조합 매칭

### 2.2 전문가 연결 및 상담 시스템
5. **약사 화상/채팅 상담**: 고객 데이터 기반 1:1 비대면 상담 예약 및 진행 (WebRTC)
6. **의사 추천 시스템**: 의료진 전용 인터페이스에서 환자에게 제품 추천 + 추천코드/UTM링크/QR코드 발급
7. **복약 관리 기능**: 소분 영양제 섭취 시간 알림(푸시) 및 복약 이력 관리

### 2.3 리워드 및 수익 분배 모델
8. **의사 추천 리워드**: 추천 링크/QR 통한 가입/구매 발생 시 리워드 지급
9. **약사 판매 수수료**: 상담 후 제품 결제 시 자동 수수료 계산·지급
10. **정산 어드민**: 전문가별 추천/상담 이력 + 수수료 정산 대시보드 (월별 수수료 추이 차트)

### 2.4 사용자 영역
11. **회원가입/로그인**: 카카오 간편가입, SNS 로그인
12. **본인인증**: 카카오뱅크, 핸드폰 간편인증, 네이버 PAY 본인인증
13. **마이데이터 연동**: DTC 결과 조회 + 건강검진 기록 불러오기
14. **AI 건강 리포트**: 건강점수, 건강상태 등급(건강해요/주의가 필요해요/질환이 의심돼요), 추천 성분, 유전자 마커별 상세(위험/주의/양호), 생활습관 개선 팁
15. **화상 상담 예약**: 약사/전문의 선택 → 예약 → 상담
16. **소분 건기식 정기구독 및 결제**: PG 결제, 정기배송, 섭취 리워드(최대 30% 할인)
17. **마켓**: 건기식 상품 목록/상세/장바구니/결제
18. **고객센터**: 마이페이지, 검사결과, 상담내역, 영양제 추천내역, 주문배송내역, 정기배송내역, 회원정보수정, 공지사항, FAQ, 1:1 문의, 이용약관

### 2.5 전문가 영역
19. **약사용**: 환자 데이터 뷰어, 화상 상담 툴, 제품 조합 구성/발송, 영양소 가이드
20. **의사용**: 추천 제품 리스트 관리, 환자용 추천 링크/QR 생성, 리워드 현황 조회, 환자 목록(검사상태/결과/추천제품/구매여부/수수료)

### 2.6 관리자(Admin) 영역
21. **제품 DB 관리**: 건기식 제품 등록/수정/삭제
22. **설문 로직 관리**: 설문 항목 및 분기 로직 설정
23. **전문가 승인/관리**: 의사/약사 가입 승인, 자격증 검증
24. **자동 정산 시스템**: 월별 수수료 자동 계산, 정산 예정일 관리
25. **배송/CS 관리**: 주문/배송 상태 관리, 고객 문의 관리
26. **건강매거진**: 콘텐츠 관리

### 2.7 개발 범위 (턴키)
27. **사용자용**: Android/iOS 앱 (하이브리드) + 마케팅용 모바일 웹
28. **전문가용**: 약사용 PC웹, 의사용 PC/모바일웹
29. **관리자용**: PC웹

### 2.8 암묵적 필요 기능
30. **앱 스토어 배포**: iOS App Store, Google Play Store 등록
31. **알림 시스템**: 복약 알림, 상담 예약 알림, 정산 알림 (FCM/APNs)
32. **검색/필터**: 상품 검색, 성분별 필터
33. **리뷰 시스템**: 맞춤형 건강식품 리뷰
34. **배송 추적**: 택배사 API 연동
35. **통계/분석**: 관리자용 서비스 통계 대시보드
36. **보안**: 개인건강정보 보호 (의료정보 보안 표준)

## 6. 리스크 분석

| 리스크 | 영향도 | 대응 |
|--------|--------|------|
| CODEF API 연동 난이도 | 높음 | 사전에 API 문서/테스트 계정 확보 필요. 스크래핑 방식이라 불안정할 수 있음 |
| 자사 DTC API 미확정 | 높음 | API 문서가 없으면 연동 불가. 착수 전 반드시 확보 필요 |

| 본인인증 3종 연동 | 중간 | 각 인증사별 계약/심사 필요. 기업 계정 필요 (개인 불가할 수 있음) |
| 앱 스토어 심사 | 중간 | 건강/의료 관련 앱은 심사 기준이 엄격. 리젝 가능성. 36일 버퍼로 대응 |
| 의료정보 보안 | 높음 | 개인건강정보 처리 시 의료법/개인정보보호법 준수 필요 |
| 경쟁 치열 | 중간 | 40건 제안 접수. 기업 개발사 우대 → 데모+PDF로 차별화 |

## 7. 데모 구현 범위 (3단계 분류)

### 완전 구현 가능 (데모에서 동작)
- 회원가입/로그인 UI (카카오 로그인 Mock)
- 메인 홈 (DTC소개, 맞춤형 건기식, 배너)
- DTC 유전자 결과 조회 화면 (더미 데이터로 12개 마커 시각화)
- 건강검진 결과 조회 화면 (더미 데이터로 13개 항목 시각화)
- AI 건강 리포트 (건강점수, 상태 등급, 추천 성분, 마커별 상세)
- 설문 시스템 UI (다단계 설문 플로우)
- 마켓 (상품 목록/상세/검색/필터/장바구니/결제 UI)
- 정기구독 UI
- 전문가 프로필/선택 화면
- 약사 상담 예약 UI
- 의사 파트너센터 (대시보드, 환자 목록, 추천 링크/QR, 수수료 현황)
- 관리자 페이지 (대시보드, 제품관리, 전문가 승인, 정산, 주문관리)
- 고객센터 (마이페이지, 내역, FAQ, 공지, 문의)
- 건강매거진
- 복약 관리 UI (섭취 알림 설정, 복약 이력)
- 모바일 반응형 (하단탭바: 홈, 마켓, 내결과, 상담, MY)

### 외부 API 필요 → Mock UI로 구현

## 8. 진행 의견

**조건부 추천**

긍정 요소:
- 예산 8,000만원은 규모 대비 적정 수준
- 기획서가 상세하여 요구사항 파악이 명확
- 유전자 마커/검진항목 매핑표가 이미 정의되어 있어 추천 알고리즘 설계 수월
- 레퍼런스(핏타민)가 명확하여 UI/UX 방향 설정 용이

주의 요소:
- **의뢰인이 "기업/법인" 지원을 우대** — 1인 프리랜서로는 선정 가능성이 낮을 수 있음
- **외부 API 5건 이상 연동** — CODEF, DTC, 본인인증 3종, PG, 화상SDK 등. 각각 계약/심사/테스트 필요
- **하이브리드 앱 + 스토어 배포** — 건강/의료 앱 심사 기준 엄격
- **공수 118일 vs 기간 110일** — 일정 여유 없음
- **제안 40건** — 경쟁 매우 치열

**착수 전 반드시 확인 필요 사항**:
1. 자사 DTC API 문서 및 테스트 환경
2. CODEF API 계약 및 테스트 계정
3. 본인인증 서비스 기업 계약 가능 여부
4. 화상상담 SDK 선택 및 비용 합의
