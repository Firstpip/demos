## 리스크

### 1. Figma 디자인 reproduce 품질
- Figma 기획 문서가 공유된 상태 — 디자인 시스템 파악 후 1:1 reproduce 필요
- 바이브코딩으로 Figma의 픽셀 단위 구현은 약간의 조정 공수 필요 (±1일)

### 2. 롤링 애니메이션 디테일
- Dooray 수준의 부드러운 스크롤 연동 롤링은 구현 난이도 중간
- GSAP ScrollTrigger 사용 시 안정적, 하지만 미세 조정에 시간 소요

### 3. 의뢰인 변동 요구사항
- "필요한 내용을 가볍게 정리" 단계 → 진행 중 기획 변경 가능성
- 착수 계약 시 수정 횟수 명시 권장 (예: 주요 피드백 2회 + 폴리싱 1회)

### 4. 세금계산서/부가세 처리
- 제안가 VAT 별도/포함 명확히 하지 않으면 정산 시 이슈

### 5. 경쟁
- 랜딩페이지 카테고리는 제안자 수 많음 예상
- 차별화: 실제 동작 데모 + 제안서 PDF

### 6. 채널톡 벤치마킹 관련
- 기획서 디자인이 채널톡 랜딩을 주 벤치마킹 대상으로 사용 — 섹션 구성·카피·카드 그리드가 유사
- 리스크: 지나치게 유사할 경우 저작권/표절 이슈 가능
- 대응: 동일 구조는 사용하되 **카피·컬러·일러스트·아이콘은 SafeWorks(건설 IT, 안전) 톤으로 재해석**. 채널톡 특유의 대화형 메시저 UI는 제거하고 건설 현장 대시보드/안전 관리 UI로 교체
