## 1. 도매꾹 (Domeggook)

- **URL**: https://domeggook.com
- **유형**: 국내 최대 B2B 도매 오픈마켓
- **특징 요약**: 개인/사업자 모두 이용 가능한 대형 도매 플랫폼. 51만+ 판매자, 대규모 상품 DB 보유. 도매매(배송대행 B2B)와 연계 운영.

### 페이지 구성 분석

| 영역 | 구성 |
|------|------|
| **상단 헤더** | 관련 서비스 링크(도매매DB, 나까마, 캠퍼스 등) → 로그인/회원가입/마이페이지 → 검색바(상품명, 상품번호, 판매자ID 검색 옵션) → 인기검색어 슬라이드 |
| **메인 네비게이션** | 전체카테고리 드롭다운(1depth → 2depth → 배너 3단계), 1688상품관, 셀러몰, 패키지배송, 최저가, 인기상품, MD추천, 기획전 |
| **메인 콘텐츠** | 대형 배너 슬라이더 + 좌/우 윙 배너 |
| **좌측 윙** | 롤링 배너(3초 자동갱신), 프로모션 |
| **우측 윙** | 판매자 순위(51만+), 고객센터(1588-0628 상시 노출), 찜한 상품, 최근 본 상품 |
| **하단** | 고객센터 정보, 사업자 정보, 이용약관 |

### 회원가입/인증 UX

- **회원 유형 3단계**: 일반회원 → 인증회원 → 기업정회원
- **일반회원**: 아이디/비밀번호/이메일 입력 → 휴대폰 인증(SMS 또는 PASS 앱) → 가입 완료
- **사업자 인증**: My페이지 → 개인정보관리 → 사업자인증신청/변경 → 사업자등록증 파일 첨부 또는 팩스 제출 → 관리자 확인 후 승인
- **특이사항**: 도매꾹은 개인도 구매 가능, 도매매는 사업자 회원만 거래 가능 (차별화 포인트)

### 상품 표시 방식

- **상품 카드**: 썸네일 이미지 + 상품명 + 도매가(단가) 표시
- **가격 표시**: 도매가 중심 (예: "2,470원"), 소매가 별도 미표시
- **재고/수량**: 백엔드에서 qty 데이터 관리, 프론트에서는 직접 노출하지 않음
- **검색 옵션**: 상품명, 상품번호, 판매자ID, 판매자닉네임으로 세분화 검색

### 주문 흐름

- 장바구니 → 주문내용확인 → 결제 (기본 흐름)
- 주문/배송관리 섹션 별도 제공
- e-money/포인트 결제 옵션 지원
- 상품문의 기능 (B2B 특성상 사전 문의 단계 강조)

### 디자인 톤

| 항목 | 상세 |
|------|------|
| **주요 컬러** | 녹색(#66C000) — 카테고리/검색 버튼 |
| **강조색** | 빨간색(#F63700) — 중요 텍스트, 가격 |
| **배경** | 흰색(#FFFFFF), 밝은 회색(#F8F8F8) |
| **텍스트** | 검은색(#1B1B1B), 회색(#999999) |
| **폰트** | Malgun Gothic, Apple SD Gothic Neo (시스템 폰트) |
| **폰트 크기** | 본문 11-12px, 제목 13-15px |
| **톤앤매너** | 정보 밀도 높음, 장식 최소화, 검색/카테고리 중심의 실용적 디자인 |

### 이 프로젝트에 참고할 포인트

- **회원 등급 체계**: 일반 → 인증 → 기업정회원 3단계 → 우리 프로젝트는 사업자 전용이므로 "가입신청 → 관리자승인 → 활성회원" 플로우 참고
- **검색 세분화**: 상품번호, 판매자 기준 검색은 B2B에서 유용
- **고객센터 상시 노출**: 전화번호 우측 고정 패턴은 B2B 신뢰 구축에 효과적
- **AI 검색 기능**: 이미지/음성 검색 배너 — 향후 확장 고려

## 2. 도매매 (Domemae)

- **URL**: https://domemae.com (실제 서비스: https://domemedb.domeggook.com)
- **유형**: B2B 배송대행 전문 플랫폼 (도매꾹 자회사)
- **특징 요약**: 사업자 전용. 공급사가 상품을 보유하고, 셀러가 판매 시 공급사가 직접 배송하는 배송대행 B2B 모델. 100만+ 전문셀러 네트워크 보유.

### 페이지 구성 분석

- 도매꾹과 유사한 구조이나 **사업자 전용**이라는 점에서 차이
- 공급사(판매자) 포털과 셀러(구매자) 포털이 분리
- 상품 카탈로그 중심 구성 — 셀러가 자신의 쇼핑몰에 등록할 상품을 소싱하는 용도

### 회원가입/인증 UX

- **사업자 회원만 가입 가능** (개인 회원 불가 — 도매꾹과의 핵심 차이)
- 사업자등록증 제출 필수
- 승인 후 상품 소싱 및 거래 가능

### 상품 표시 방식

- 도매가(공급가) 중심 표시
- 상품 이미지 무상 제공 (셀러가 자사 쇼핑몰에 바로 등록할 수 있도록)
- 상품 데이터 API/연동 제공

### 주문 흐름

- 셀러 → 도매매에서 상품 소싱 → 자사 쇼핑몰에 등록 → 고객 주문 발생 → 도매매 공급사가 직배송
- **배송대행 모델**: 재고 없이 판매 가능

### 수수료 구조

- 판매수수료: 3% ~ 6.3% (VAT 별도, 신용카드수수료 포함)
- 최저수수료과금 없음 — 저가 상품에 유리
- 오픈마켓 중 최저 수수료 수준

### 디자인 톤

- 도매꾹 계열 플랫폼으로 유사한 녹색 기반 컬러 체계
- 기능 중심의 실용적 UI
- 정보 밀도 높은 리스트형 레이아웃

### 이 프로젝트에 참고할 포인트

- **사업자 전용 가입 모델**: 우리 프로젝트의 핵심 요구사항과 일치 — 가입 시 사업자등록증 필수 제출 → 관리자 승인 플로우
- **배송대행 모델**: 직접 참고는 아니지만, 공급사/셀러 역할 분리 구조는 참고 가능
- **상품 데이터 연동**: ERP 연동 시 상품 데이터 동기화 방식 참고

## 3. 온채널 (OnChannel)

- **URL**: https://www.onch3.co.kr
- **유형**: B2B 유통 전문 플랫폼 (도매/폐쇄몰/종합몰/전문몰/공동구매)
- **특징 요약**: 공급사와 판매사를 연결하는 위탁판매 도매 플랫폼. 50만+ 상품 DB. 폐쇄몰/종합몰 채널 분리 운영 지원.

### 페이지 구성 분석

| 영역 | 구성 |
|------|------|
| **상단** | 공급사/판매사 분리 메뉴, 로그인/회원가입 |
| **메인 콘텐츠** | 대형 배너, 상품 카테고리, 기획전시관 |
| **큐레이션** | MD PICK, 베스트 상품, 최저가 인증상품, 단독상품관 |
| **부가 기능** | 판매랭킹, 셀러대학(교육 프로그램), 챌린지 프로그램 |

### 회원가입/인증 UX

| 회원유형 | 가입 절차 | 승인 방식 |
|----------|----------|----------|
| **사업자** | 사업자등록증 제출 | 가입 즉시 승인 |
| **교육생** | 교육 과정 등록 | 관리자 승인 필요 |
| **해외 사업자** | 해외 사업자 정보 제출 | 관리자 승인 필요 |

### 상품 표시 방식

- 도매가 중심 표시
- 카테고리별 상품 분류
- MD PICK, 최저가 인증 등 큐레이션 태그
- 매일 업데이트되는 상품 추천 + 시즌/트렌드 기획전

### 주문 흐름

- 위탁판매 모델 중심
- 주문수집/발주 자동화 기능
- 월 2회 정산 시스템 (클릭 방식 간편 정산)
- 비즈니스 관리 대시보드 제공

### 폐쇄몰 관련 기능 (핵심 참고)

- **채널 분리 운영**: 폐쇄몰, 종합몰, 공동구매 채널을 분리 운영 가능
- **가격 정책 선택**: 가격 자율 vs 가격 준수 채널 선택 가능
- **폐쇄몰 특성**: 회원에게 일반 소비자보다 낮은 가격 제공, 특별 프로모션/서비스 제공

### 디자인 톤

| 항목 | 상세 |
|------|------|
| **브랜드 컬러** | 그린(#10AE5A) 기반 |
| **레이아웃** | 대형 배너 + 아이콘 중심의 명확한 정보 계층 |
| **톤앤매너** | 캐주얼하면서도 전문성 강조, 모던한 느낌 |

### 이 프로젝트에 참고할 포인트

- **폐쇄몰 채널 분리 운영**: 우리 프로젝트의 "회원별 상품 공개" 요구사항과 직접 연결
- **회원 유형별 승인 프로세스 차이**: 사업자 즉시승인 vs 관리자 승인 — 우리는 관리자 승인 필수이므로 교육생/해외사업자 플로우 참고
- **가격 정책 분리**: 회원별 할인 구현 시 가격 자율/준수 모델 참고
- **주문수집/발주 자동화**: ERP 연동 시 참고할 패턴
- **정산 시스템**: 월 2회 정산 패턴

## 4. Faire

- **URL**: https://www.faire.com
- **유형**: 글로벌 B2B 도매 마켓플레이스
- **특징 요약**: 독립 브랜드와 리테일러 연결. 50만+ 리테일러, 10만+ 브랜드. Net 60 결제, 무료 반품 등 혁신적 B2B 거래 조건. 소비자 수준의 UX를 B2B에 적용한 대표 사례.

### 페이지 구성 분석

| 영역 | 구성 |
|------|------|
| **헤더** | 깔끔한 로고 + 검색바 + 카테고리 + 로그인 |
| **메인** | 큐레이션 기반 상품 발견 (트렌드별, 카테고리별) |
| **상품 페이지** | 고품질 이미지 + 명확한 도매가 + MOQ(최소주문수량) + 케이스/단품 옵션 |
| **주문 관리** | 배송 추적, 주문 이력, 브랜드 커뮤니케이션 |
| **대시보드** | 판매 분석, 트래픽, 트렌드 인사이트 |

### 회원가입/인증 UX

**리테일러(구매자)**:
1. 사업자등록증, 세금ID 제출
2. 매장 사진 포함 사업 프로필 작성
3. 활성 소매 운영 증명
4. 신용 이력 검증 (Net 60 조건 적용 시)

**브랜드(판매자)**:
1. 사업 등록 및 세금 정보 제출
2. 고품질 이미지 포함 제품 카탈로그 제출
3. 도매가격 구조 설정
4. 5~7일 내 주문 이행 능력 증명

### 상품 표시 방식

| 항목 | 방식 |
|------|------|
| **가격** | 도매가 중심, 소매가의 50~60% 수준, 투명한 가격 표시 |
| **MOQ** | 유연한 최소주문 ($100~$500), 전통 도매($1,000+) 대비 낮음 |
| **옵션** | 케이스(풀박스) vs 단품(싱글 유닛) 선택 가능 |
| **이미지** | 고품질 상품 사진 필수, 브랜드 스토리텔링 포함 |
| **검색** | 카테고리, 가격, 배송지역 등 다중 필터 |

### 주문 흐름

1. 상품 검색/발견 → 카트 추가 → 체크아웃 → 결제 → 브랜드 직배송 → 배송 추적

**결제 방식**:
| 방식 | 설명 |
|------|------|
| **Net 60** | 배송 후 60일 후 결제 (적격 리테일러) |
| **신용카드** | 즉시 결제 |
| **Faire Credit** | 적격 리테일러용 연장 결제 |

**반품 정책**: 첫 주문 무료 반품 — 리스크 최소화로 신규 브랜드 시도 유도

### 디자인 톤

| 항목 | 상세 |
|------|------|
| **컬러** | 화이트 기반 + 블랙 텍스트 + 브라운/베이지 액센트 |
| **폰트** | 모던 산세리프, 깔끔한 타이포그래피 |
| **레이아웃** | 넉넉한 여백, 그리드 기반, 이미지 중심 |
| **톤앤매너** | 프리미엄 + 미니멀, 소비자 쇼핑몰 수준의 세련됨 |
| **핵심 차별점** | "B2B인데 B2C처럼 예쁘다" — 전문성과 감성의 균형 |

### 이 프로젝트에 참고할 포인트

- **소비자 수준 UX를 B2B에 적용**: 기능 중심의 투박한 B2B UI가 아닌 세련된 디자인 지향
- **사업자 인증 + 신용 검증 2단계**: 가입 시 사업 증명 + 결제 조건에 따른 신용 검증
- **유연한 MOQ**: 최소주문 기준을 상품/브랜드별로 유연하게 설정
- **Net 60 결제 모델**: 우리 프로젝트는 계좌이체/무통장이지만, 후불 결제 개념은 참고 가능
- **Faire Direct**: 기존 고객을 플랫폼에 초대하는 기능 — 관리자 직접 주문 생성과 유사한 맥락
- **추천 엔진**: 상품 발견 → 회원별 상품 공개와 연결 가능

## 6. 디자인 시스템 추천

### 6.1 컬러 팔레트

B2B 폐쇄몰은 **전문성**, **신뢰감**, **비즈니스 톤**이 핵심입니다.

```
Primary (주요 컬러)
├── Navy Blue: #1B3A5C — 메인 컬러, 신뢰감/전문성
├── Dark Navy: #0F2440 — 헤더, 사이드바
└── Light Navy: #2E5C8A — 호버, 활성 상태

Secondary (보조 컬러)
├── Teal: #0D9488 — 액센트, CTA 버튼, 성공 상태
└── Light Teal: #14B8A6 — 호버 상태

Neutral (중립 컬러)
├── Gray 900: #111827 — 본문 텍스트
├── Gray 600: #4B5563 — 보조 텍스트
├── Gray 400: #9CA3AF — 비활성 텍스트, 플레이스홀더
├── Gray 200: #E5E7EB — 구분선, 테두리
├── Gray 100: #F3F4F6 — 배경 (카드, 섹션)
└── White: #FFFFFF — 기본 배경

Semantic (의미 컬러)
├── Success: #059669 — 승인, 완료
├── Warning: #D97706 — 주의, 대기 중
├── Error: #DC2626 — 오류, 거부
└── Info: #2563EB — 정보, 알림

Price (가격 전용)
├── Sale Price: #DC2626 — 할인가, 특가
└── Original Price: #9CA3AF — 정가 (취소선)
```

**컬러 선정 이유**:
- Navy Blue: 금융/비즈니스 서비스에서 가장 많이 사용되는 신뢰 컬러. 도매꾹/온채널의 Green 대비 더 B2B 전문몰에 적합
- Teal 액센트: Navy와 조화로우면서도 CTA에서 눈에 띄는 포인트 컬러
- 넓은 Gray 스펙트럼: 정보 밀도가 높은 B2B UI에서 계층 구조 표현에 필수

## 7. 프로젝트 적용 시사점

### 핵심 설계 원칙

1. **폐쇄성 우선**: 로그인 전에는 어떤 상품/가격 정보도 노출하지 않음 (Faire와 유사하되 더 엄격)
2. **사업자 전용 신뢰 UI**: Navy Blue 기반의 전문적 톤, 사업자등록번호 자동 검증
3. **정보 밀도 vs 심미성 균형**: 도매꾹의 정보밀도 + Faire의 디자인 감성 결합
4. **데스크톱 우선, 모바일 대응**: B2B 사용자 특성 반영

### 페이지별 설계 방향

| 페이지 | 참고 서비스 | 설계 방향 |
|--------|------------|----------|
| **로그인/게이트** | 자체 설계 | 미니멀한 로그인 화면, 사업자 전용 안내, 회원가입 유도 |
| **회원가입** | 도매매 + 온채널 | 단계별 폼 (기본정보 → 사업자정보 → 약관), 사업자등록증 업로드 |
| **메인(로그인 후)** | Faire + 도매꾹 | 배너 + 추천상품 + 카테고리 + 최근주문 빠른접근 |
| **카테고리/목록** | 도매꾹 | 좌측 필터 사이드바 + 그리드/리스트 전환 + 정렬 옵션 |
| **상품 상세** | Faire | 고품질 이미지 + 도매가 + 옵션/재고 + 최소주문 + 구매제한 표시 |
| **장바구니** | Faire | 수량 조절 + 옵션 변경 + 최소주문 검증 + 합계 |
| **주문/결제** | 자체 설계 | 무통장입금/계좌이체 선택 + 배송정보 + 주문 요약 |
| **마이페이지** | 온채널 | 주문내역 + 회원정보 + 할인등급 확인 |
| **관리자** | 온채널 + 자체 | 회원관리(승인) + 상품관리 + 주문관리 + 직접주문생성 + 팝업관리 + ERP 연동 설정 |

### 기술 참고사항

- **ERP 연동**: REST API 기반 상품/주문/재고 동기화 (온채널의 주문수집/발주 자동화 참고)
- **회원별 상품 공개**: DB 레벨에서 회원-상품 매핑 테이블 설계 (온채널의 채널 분리 개념)
- **회원별 할인**: 회원 등급 테이블 + 등급별 할인율 관리 (Faire의 가격 구조 참고)
- **구매 제한**: 상품별 최소금액/최소수량/판매기간 필드 + 주문 시 서버사이드 검증
- **관리자 주문 생성**: 관리자 전용 API 엔드포인트, 일반 주문과 동일 플로우 + 회원 대리 선택 기능
