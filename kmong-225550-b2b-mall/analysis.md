## 2. 선정 기술스택

| 영역 | 기술 | 선정 이유 |
|------|------|----------|
| 프레임워크 | Next.js 14+ (App Router) | 프론트+백엔드 통합, API Routes로 ERP/회계 연동 엔드포인트 구현, 1인 개발 생산성 최고 |
| 스타일링 | Tailwind CSS | 반응형 기본 지원, B2B 깔끔한 UI 빠르게 구현 |
| DB | PostgreSQL (AWS RDS) | ERP 연동 데이터 JSON 지원, 대시보드 집계 쿼리 성능, 관리형 DB로 백업/복구 자동화 |
| ORM | Prisma | 타입 안전성, 마이그레이션 도구 내장 (아임웹 데이터 이관 시 스키마 관리) |
| 인프라 | AWS Lightsail | 고정 비용 VPS, cron job으로 ERP 동기화/회계 자동발행 처리, 서버 프로세스 시간 제한 없음 |
| 파일 저장 | AWS S3 | 사업자등록증/소매인지정서 업로드, presigned URL 직접 업로드, CloudFront CDN 연동 가능 |
| 인증 | NextAuth.js + 자체 인증 | 사업자 인증 + 본인인증(NICE/PASS) 이중 체계, 서버사이드 callback 처리 |
| 엑셀 | exceljs | 서버사이드 엑셀 생성, 필터/스타일링 지원, 스트리밍 출력으로 대용량 대응 |
| 결제 | 없음 (PG 불필요) | 계좌이체/무통장 입금만 → 관리자 수동 확인 |

## 12. 데이터 구조

### Member (회원)
```
{
  id, businessName, businessNumber, ownerName,
  email, phone, password,
  businessLicense (file URL), retailDesignation (file URL),
  verificationStatus (pending | approved | rejected),
  memberGroup (standard | vip | premium),
  cashReceiptNumber, discountRate,
  purchaseHistory, approvedAt, approvedBy
}
```

### Product (상품)
```
{
  id, name, category, description, images[],
  options[{ name, values[], stock[] }],
  basePrice, visibleToGroups[],
  purchaseLimit { minAmount, maxQuantity, periodDays },
  isActive, createdAt
}
```

### Order (주문)
```
{
  id, memberId, items[{ productId, optionValues, quantity, price }],
  totalAmount, discountAmount, finalAmount,
  status (pending | confirmed | shipping | delivered | cancelled),
  paymentMethod (bankTransfer | deposit),
  paymentStatus (pending | confirmed),
  cashReceiptIssued, taxInvoiceIssued,
  createdBy (member | admin),
  isEditable, createdAt, confirmedAt
}
```

### Popup (팝업)
```
{
  id, title, type (single | slide),
  images[{ url, link }],
  startDate, endDate, isActive
}
```
